数据库是整个系统的地基。它由25张"表"构成，每张表就像一个专门用途的登记本——有的记设备，有的记客户，有的记每一笔钱的流动。所有表通过唯一编号（UUID）互相关联，追踪一台叉车的完整历史，就像拉一条线，能串起它从出厂到报废经历的每一件事。
数据只存一次，不重复 状态变化全程记录不删除 计算结果不存库，实时算 每条记录必须知道从哪来
id
每条记录的唯一身份证号（UUID格式）
系统自动生成，永远不重复
created_at
这条记录创建的精确时间
系统自动写入，不可修改
source
数据从哪里来
人工录入 / 系统生成 / 供应商文件 / API导入
confidence
这条数据有多可靠
已核验 / 有文件 / 人工录入 / 系统推断
以上四个字段每张表都有，是数据底座的基础规则。下面不再重复列出。
UUID · 唯一标识符
TEXT · 文字内容
数字 · 金额/数量
DATE · 日期时间
BOOL · 是/否
ENUM · 固定选项
JSON · 灵活数据
金色字段 = 主键（这张表的身份证）
蓝色字段 = 外键（指向另一张表）
绿色字段 = 元字段（四个基础字段）

智能视图 · Smart Views
5个预建视图（智能层的数据接口）
视图是从原始数据表里实时计算出来的"汇总视角"。智能层（AI）读取这些视图，不直接读底层表。视图里的所有计算值都是实时的，永远最新。
v_equipment_current
当前在库设备视图
把设备主表、机型表、供应商表、法人实体表的信息合并，展示每台活跃设备的完整快照。排除已归档、已退回、已报废的设备。
用途：库存看板 · 销售查车
v_ar_aging
应收账款账龄分析
实时计算每笔应收账款逾期多少天，自动归入账龄桶（当期 / 1-30天 / 31-60天 / 61-90天 / 90天以上）。账龄天数不存库，实时算。
用途：财务催款 · 坏账预警
v_hire_expiring
60天内到期租赁合同
筛选出60天内即将到期的所有租赁合同，附带客户联系方式和负责销售信息，让系统可以自动触发续租跟进。
用途：续租自动跟进 · 设备回收安排
v_ap_due_soon
30天内到期应付账款
筛选出30天内需要支付的供应商账款，包括金额、到期日、供应商信息，让财务提前安排付款，避免逾期影响供应商关系。
用途：账期管理 · 供应商关系维护
v_po_pending
PO待办池
展示所有全款已到账、等待采购负责人下PO的销售订单。包括等待天数，让采购知道哪些已经等太久了，需要优先处理。
用途：采购待办 · 流程提速
